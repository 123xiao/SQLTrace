# 日志执行链路可视化工具

一个基于 React 的日志分析和 SQL 执行链路可视化工具，帮助开发者快速理解日志中的 SQL 执行流程和数据库表结构。

## 功能特性

### 核心功能
- **日志解析**：自动解析日志文件，提取 SQL 语句和执行信息
- **SQL 可视化**：以流程图形式展示 SQL 执行链路
- **表结构管理**：维护数据库表结构和字段注释
- **智能分析**：自动识别 SQL 中使用的表和字段，并关联表结构信息

### 界面特性
- **主题切换**：支持白天/夜间模式切换
- **表分组**：按表名前缀自动分组（如 ACT_、SYS_ 等）
- **搜索过滤**：支持表名、字段名、注释的快速搜索
- **高亮显示**：搜索结果高亮，便于快速定位
- **模态详情**：点击表名查看完整表结构和字段信息

## 使用指南

### 1. 导入日志文件
点击"选择日志文件"按钮，选择包含 SQL 语句的日志文件。支持的日志格式：
- 包含 SQL 关键字（SELECT、INSERT、UPDATE、DELETE）的文本文件
- 每行一条日志记录

### 2. 配置表结构
在"表结构管理"标签页中：
- 点击"添加表"按钮新增表定义
- 填写表名、表注释
- 添加字段信息（字段名、类型、注释）
- 表结构自动保存到浏览器本地存储

### 3. 分析 SQL
在"SQL 分析"标签页中：
- 粘贴或输入 SQL 语句
- 点击"分析 SQL"按钮
- 查看涉及的表和字段信息
- 点击表名查看完整表结构

### 4. 查看执行链路
在"执行链路"标签页中：
- 查看日志中所有 SQL 的执行顺序
- 点击 SQL 节点查看详细信息
- 使用搜索功能快速定位特定 SQL

## 数据存储

所有数据存储在浏览器的 LocalStorage 中：
- `dbSchema`：数据库表结构定义
- `logContent`：最近导入的日志内容

## 常见问题

**Q: 日志文件格式有什么要求？**
A: 日志文件应为文本格式，每行包含一条日志记录，SQL 语句需包含 SELECT、INSERT、UPDATE 或 DELETE 关键字。

**Q: 表结构数据会丢失吗？**
A: 表结构保存在浏览器 LocalStorage 中，清除浏览器数据会导致丢失。建议定期导出备份。

**Q: 支持哪些数据库？**
A: 工具本身不连接数据库，支持所有标准 SQL 语法的分析。
